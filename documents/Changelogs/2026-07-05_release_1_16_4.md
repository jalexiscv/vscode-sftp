# Release 1.16.4: identidad del fork y paquete instalable

**Fecha:** 2026-07-05
**Área:** build

## Descripción

Primera release del fork corregido. Se actualiza la identidad del paquete
para la distribución propia y se genera el `.vsix` instalable
(`sftp-1.16.4.vsix`, publicado como GitHub Release en
[jalexiscv/vscode-sftp](https://github.com/jalexiscv/vscode-sftp/releases)).

## Tipo de Cambio

- `Cambiado`

## Archivos Afectados

### [MODIFICADO] `package.json`
- `version` 1.16.3 → 1.16.4; `publisher`/`author`/`bugs`/`homepage`/`repository` apuntan a jalexiscv/vscode-sftp. El ID de la extensión pasa a ser `jalexiscv.sftp`.

### [MODIFICADO] `CHANGELOG.md` (raíz)
- Entrada 1.16.4 con el resumen de las correcciones de esta iteración (en inglés, siguiendo el formato del upstream), enlazando los issues/PRs originales.

### [MODIFICADO] `README.md`
- Nota de fork corregido al inicio, conservando la atribución a Natizyskunk y liximomo.

## Impacto

- La extensión es instalable desde el `.vsix` (`code --install-extension sftp-1.16.4.vsix`).
- **Advertencia**: la extensión registra los mismos comandos `sftp.*` que la original; hay que desinstalar o deshabilitar la SFTP de Natizyskunk antes de instalar esta para evitar conflictos.
- Para publicar en el Marketplace haría falta crear el publisher `jalexiscv` en el portal de VS Marketplace.
